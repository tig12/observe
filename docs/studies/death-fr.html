Deaths in France 1970 - 2025

This study is based on data coming from www.data.gouv.fr/datasets/fichier-des-personnes-decedees, which contains nominative data of persons deceased in France since 1970 (with birth day and death day, but untimed).
This study concerns persons who died in France between 1970 and 2025 and contains 28 803 832 persons.

The raw data

The data used for this study can be downloaded from data.gouv.fr.
The download and build of a local sqlite database (called death-fr.sqlite3), used as the raw data of this study, is handled by another program called g5 (see github.com/tig12/g5).
If you want to build death-fr.sqlite3 on your own machine, you can use g5 code, documented on github.com/tig12/g5/tree/main/src/commands/enrich/deathfr

The result of the g5 command used to build death-fr.sqlite3 gives informations about the number of lines inserted: 
php run-g5.php enrich deathfr raw2sqlite 1970-2025 > data/tmp/enrich/death-fr/sqlite-build-report.log

-------------------------------------------------------
Total Execution time: 547.3 s - 00:12:07
-------------------------------------------------------
28 917 511 lines parsed
28 803 832 lines inserted
----------------------- ERRORS ------------------------
ERR_NAME:       67 incorrect name - inserted anyway
ERR_BDAY:       112 808 incorrect birth day - not inserted
ERR_DDAY:       798 incorrect death day - not inserted
ERR_POSTERIOR:  71 birth posterior to death - not inserted
ERR_EXCEPTION:  2 exception
=> skipped      113 677 lines because of date problem
This shows that the files of data.gouv.fr (1970 - 2025) contain 28 917 511 dates but around 114 000 lines are incorrect.
So the input data for this study contain 28 803 832 persons.
As each person has a birth day and a death day, the study deals with 57 835 022 dates.

death-fr.sqlite3

death-fr.sqlite3 contains one table, person with this structure: 
create table person(
    fname varchar(80),
    gname varchar(80),
    sex character(1),
    bday character(8),
    bcode character(5),
    bname character(30),
    bcountry varchar(80),
    dday character(8),
    dcode character(5),
    dact character(9)
);
create index idx_bday ON person(bday);
